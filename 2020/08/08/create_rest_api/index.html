Open APIでRest APIのソースコードとAPI定義書を自動生成

2020-08-08

#OpenAPI #SpringBoot #Java #RestAPI

要約

  • Open APIはRest APIを作成する上で何かと便利
    • Rest APIをYAML形式で簡単に定義できる
    • 定義したYAMLファイルからRest APIのソースコードを生成できる
    • 定義したYAMLファイルからはAPI定義書(静的HTML)も生成できる

前提

  • 環境： Windows 10
  • 言語： Java (Spring)

手順

  • OpenAPIのgithubのサンプル(ペットショップ) をベースにRest APIを作成する

  • Spring InitializerからSpring Bootアプリの雛形をダウンロード

    • 下記の通り設定を変更(指定なしはデフォルトのまま)
      • Project: Gradle projectを指定(好み)
      • Project Metadata.Java: 8
      • Dependencies: Spring web
    • Generateをクリックし、ダウンロードしたzipファイルを解凍

  • 解凍したサンプルアプリを編集

    • プロジェクトのルートディレクトリにpetstore.yamlを追加

    • build.gradleを編集

    • com.example.demo.DemoApplication.java を削除

    • .openapi-generator-ignoreというファイルをルートディレクトリに追加し、以下の内容を追記

    • ソースコード生成

    • build.gradleを編集し、生成したソースコードが依存するライブラリを追加

      • プロジェクトのルートディレクトリにpom.xmlが生成されるので、それを参考に依存を追加

      • ※以下のコマンドでpom.xmlはbuild.gradleに変換できる

      • アプリを実行

      • ブラウザからアプリにアクセスし、Rest APIが叩けることを確認する

        • http://localhost:8080
        • swagger-uiからペットショップのRest APIを叩いてみる(レスポンスステータスとして501 Not Implemented が返ってくる)

  • API定義書を生成できる
