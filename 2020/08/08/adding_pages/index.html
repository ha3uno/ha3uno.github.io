Hexoで静的HTMLのページを追加する

2020-08-08

#Hexo #Github

要約

  • Hexoでは、/blog_name/source/以下にMarkdownファイルを置いてデプロイすると、静的HTMLに変換されてサーバ上にホスティングされる
    • Markdownファイルのファイル名がそのままHTMLのファイル名となる
    • /blog_name/source/以下のディレクトリ構成がそのままサーバ上にホスティングされる
      • Github Pagesの場合、*/blog_name/source/some_path/some_dir/hoge.md* は https://[owner名].github.io/some_path/some_dir/hoge.html に配置される
  • 一方、/blog_name/source/_posts/以下にMarkdownファイルを配置すると、ブログの記事として扱われる
    • URLのパスは、作成したMarkdownファイルのFront-matterブロックにおけるdateフィールドの値をベースに決まる
    • URLの最終パスはMarkdownファイル名となり、静的HTMLはindex.htmlとして生成される
      • Markdownファイルのdateフィールドが date: 2020-08-08 13:32:50 の場合、https://[owner名].github.io/2020/08/08/[Markdownファイル名]/index.html としてホスティングされる
  • 詳細はHexoのドキュメントを参照
