流行りのClaude codeを試してみた所感


やったこと

  • サンプルのECサイトを構築
    • https://omiseyasan.onrender.com/
    • https://github.com/ha3uno/omiseyasan
      • コード部分は9割以上claudeの生成物であり、手を加えていない
    • 商品の閲覧、カート追加、アカウント登録、注文、注文履歴参照機能を持つ
    • 管理機能としてサイトの更新履歴登録、閲覧機能も

事前準備

  • geminiを使えるようにする
  • claudeのproプラン(20ドル)契約
  • vs codeでclaudeを使えるように

how-to

  • geminiにサンプルのecサイトを構築できるやり方を聞く
    • Renderというサービスで無料でwebアプリを構築できることを知る
    • 技術スタックはfrontはreact, backはgoでテキトーに
    • 単一リポジトリで、go側でfrontのhtmlとかもホスティングさせる形に
  • renderでアカウントを作成し、blueprintでアプリを、あとはdbでpostgresqlのdbを手でポチポチ構築
    • renderはgitのリポジトリと連携させ、リポジトリの指定ブランチにpushされたらrender.ymlでci/cdさせるように
  • geminiにclaudeに対する命令文を生成させつつ、ecサイトの基本的な機能を構築
    • 命令した内容は下記参照
      • https://omiseyasan.onrender.com/update-history

所要時間

  • 約8時間程度(基本的なガワ作成だけであれば3時間半でOK)

良かった点

  1. 短時間でサイト構築できた
    • 非エンジニアがモックをつくるのによさそう
  2. リファクタとかはかなりいい感じにやってくれた(main.goの分解作業)
  3. わからんがデザインとかもなんとなく最低限わかるような形で作ってくれた
  4. (goだからか)生成されるコードもわりと秩序があった

つらみ

  • claudeへの命令は連打するとtokenが溶けて制限到達して数時間利用不可になる
  • 手元のボロpc(intelチップのmac)だとつらい
  • 最初の機能のガワ追加はいいが、エラー出て動かないあたりからgeminiがアホになって頼れなくなる
    • 途中からclaudeに直接命令するスタイルに変更
  • 途中から生成物がエラーが動かなくなっていった(命令文にテスト実行も含ませるようにしたら解決)
  • db疎通、ciのエラー、環境構築まわりなどトラシュー系は人の手が必要
    • 仕組みができてればコード追加はラクにやってくれそう
  • このブログが見づらい(白っぽい背景に白文字…claudeでなんとかしたい)
